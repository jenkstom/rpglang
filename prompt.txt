You are an expert compiler engineer specializing in legacy programming languages and modern LLVM infrastructure. Your task is to build a self-contained, working RPG II compiler for Linux that uses LLVM as its backend. The compiler should compile valid RPG II source code into executable Linux binaries.

### 1. Target Language Specification (RPG II)

Note: the full RPG user manual is available in the c0918180.pdf file.

The compiler must support standard RPG II column-oriented syntax and the implicit RPG program cycle. Focus on these core components:
- File Description Specifications (F-specs): Column 6 = 'F'. Define Device files (DISK, PRINTER).
- Input Specifications (I-specs): Column 6 = 'I'. Define record layouts, fields, and positions.
- Calculation Specifications (C-specs): Column 6 = 'C'. Support operations: ADD, SUB, MULT, DIV, Z-ADD, SETON, SETOFF, COMP, GOTO, TAG, and MOVE.
- Indicators: Support resulting indicators (e.g., 01-99, LR for Last Record) for conditional C-spec execution.
- The RPG Cycle: Implement the implicit loop (Fetch record, set indicators, execute total calculations, execute detail calculations, output).

### 2. Architectural Requirements
- Frontend Language: Write the compiler in C++ or Rust (choose the one with the best-maintained LLVM bindings).
- Lexer/Parser: Implement a fixed-column parser (since RPG II relies on exact column positions rather than tokens separated by whitespace).
- Intermediate Representation (IR): Generate clean, optimized LLVM IR.
- Runtime Library: Create a small C-based runtime library (`librpgruntime.a`) to handle file I/O (reading text/EBCDIC/ASCII files sequential/indexed) and standard RPG built-in behaviors.

### 3. Incremental Build Plan (Execute Phase by Phase)
Do not write the whole compiler at once. Build it iteratively, verifying each step:

- Phase 1: Environment & Boilerplate. Set up the project structure, build system (CMake or Cargo), and link LLVM. Create a basic CLI that accepts an input file.
- Phase 2: C-Spec Math & Indicators. Parse a minimal C-spec file handling only fixed variables, `ADD`, `Z-ADD`, and `SETON LR`. Generate LLVM IR that can compile to an executable that returns an exit code.
- Phase 3: The RPG Cycle & I/O. Implement a basic F-spec and I-spec parser. Generate the implicit main loop in LLVM IR to read a sequential text file line by line, mapping columns to variables.
- Phase 4: Full C-Spec Operations. Implement `COMP`, `GOTO`, indicator conditioning (columns 9-17), and `MOVE`.
- Phase 5: Code Generation & Linking. Ensure the compiler automatically invokes `llc` and `clang`/`gcc` to link the generated object file with the runtime library into a final Linux ELF binary.

- Include a basic 'Hello World' style RPG II test program and a test script to verify that the compiler successfully parses, compiles, and runs it on Linux.
