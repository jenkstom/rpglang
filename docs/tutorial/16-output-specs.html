Chapter 10 — Output Specifications (O-specs)

↑ Table of contents

In this chapter

The O-spec (column 6 = O) describes what the program outputs — declaratively, not imperatively. Instead of writing print() or printf statements, you describe output lines: which file they go to, what type (heading, detail, total, exception), when they fire (conditioned by indicators), and what fields or constants appear at what positions. The program cycle handles the actual printing at the right time.

10.1 The "you already know" analogy

In a modern language, generating a report means imperative format strings embedded in your logic:

print(f"{'Amount':<20}{amt:>10}")
print(f"{'Total':<20}{total:>10}")

You write the print call, construct the format string, and decide when to call it. In RPG II, you declare the report layout as O-specs and the cycle emits them at the right time. You describe what the line looks like and when it should print; the language handles the printing.

This is familiar ground. O-specs are a declarative report template — similar to an HTML template or a SQL SELECT with formatting. You do not write print() calls. You declare "at detail time, print a line with the constant 'Amount =' ending at column 20, followed by the field AMT ending at column 30." The cycle does the rest.

10.2 Two line types: record lines and field lines

Like I-specs, O-specs come in two kinds of line:

  • Record line — starts a new output line. It carries the filename (cols 7–14), the type (col 15: H/D/T/E), spacing and skipping (cols 17–22), and conditioning indicators (cols 23–31).
  • Field line — describes one field or constant on the current line. Cols 7–31 are blank; the field name (cols 32–37), edit code (col 38), end position (cols 40–43), and constant (cols 45–70) define what prints and where.

One record line can be followed by many field lines — each adds another field or constant to the same printed line. The record line controls when the line prints; the field lines control what is on it.

10.3 Output types (col 15)

The record line's type column (col 15) determines when in the program cycle the line prints. This is the O-spec's equivalent of choosing "when to call print()" — but the cycle manages the timing for you:

TypeLetterWhen it fires
HeadingHAt heading time — typically once at program start (conditioned by the 1P indicator) or on page overflow
DetailDAfter detail calculations, once per record
TotalTAt total time — on control breaks (L1–L9) and at LR (end-of-file)
ExceptionEOn demand via the EXCPT operation in C-specs (not cycle-driven)

The three cycle-driven types — H, D, T — correspond to the three phases of the program cycle described in Chapter 3. Heading lines print before detail lines; detail lines print once per record; total lines print when a control break fires or at end-of-file. Exception output is the escape hatch: it prints whenever your C-spec code explicitly requests it.

10.4 Spacing and skipping (cols 17–22)

Two pairs of columns control vertical positioning on the page:

FieldColsEffect
Space Before17Number of blank lines (0–3) before this line prints
Space After18Number of blank lines (0–3) after this line prints. If cols 17–22 are all blank, the default is single-space after.
Skip Before19–20Advance to an absolute line number (01–99) before printing. A skip to a line at or before the current position starts a new page.
Skip After21–22Advance to an absolute line number after printing

The most common setting is a single digit in col 18 — "space after this many lines." For example, D 1 on a record line means type D (detail), space 1 after (one blank line between detail rows). A total line with T 2 double-spaces for visual separation from the detail rows above:

10.5 Conditioning indicators (cols 23–31)

Both record lines and field lines carry conditioning indicators in cols 23–31 — three 3-column groups in the same format as C-specs (see Chapter 9). They gate whether the line (or individual field) prints:

  • Record lines: if the conditioning indicators are not satisfied, the entire line — including all its field lines — is skipped.
  • Field lines: per-field conditioning gates individual fields. A field whose conditions do not hold is omitted, while the rest of the line still prints.

The 1P (first-page) indicator is the standard way to print headings once at the start. It turns ON before the first record and OFF immediately after, so a heading conditioned on 1P prints exactly once:

10.6 Field name, end position, and constants

Field lines use three column groups to place content on the printed line:

FieldColsDescription
Field Name32–37The field to print. Its value is placed right-justified ending at the end position.
Edit Code38Formatting code for numeric fields (commas, decimals, signs — see Chapter 17)
Blank After39B resets the field to blanks/zeros after printing
End Position40–43The rightmost output column this field occupies. If blank, the field packs immediately after the previous field.
Constant / Edit Word45–70A quoted string like 'Total:'. When present, the field-name columns are blank.

The key concept is end position. Fields and constants are right-justified to the end position — they end at that column and extend leftward. A field ending at column 30 with a 3-digit value places the last digit at column 30, the middle digit at 29, and the first digit at 28.

Constants, by contrast, are left-justified ending at the end position — they start from the left and their last character lands at the end position. So 'Amount =' ending at column 20 places the = at column 20 and the A at column 13.

The 20 for the constant sits at cols 42–43 (within the 40–43 end-position field, right-justified). The constant itself starts at col 45 with the opening quote. For the AMT field line, the field name occupies cols 32–34, and the end position 30 sits at cols 42–43.

10.10 Putting it together — a complete report

Here is a complete O-spec section for a simple report: a heading that prints once, a detail line per record showing each amount, and a grand total at end-of-file. It combines headings, constants, fields, conditioning indicators, and spacing:

Walk through the execution: at program start, 1P turns ON and the heading prints — the constant 'SALES REPORT' right-justified ending at column 40. Then 1P turns OFF. For each input record, the detail line prints: 'Amount =' ending at column 20, then AMT ending at column 30, single-spaced. When all records are exhausted, LR turns ON and the total line prints with a double-space: 'Grand total =' at column 20, then TOTAL at column 30.

The result, for three records with amounts 42, 17, and 3:

                              SALES REPORT
         Amount =                   42
         Amount =                   17
         Amount =                    3

         Grand total =              62

No print() calls. No format strings. No loops over records. The F-spec declared the file, the I-spec parsed each record into AMT, one C-spec accumulated TOTAL, and the O-specs described the layout. The program cycle tied it all together.
