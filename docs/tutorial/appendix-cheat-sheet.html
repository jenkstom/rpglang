How to read these cards

Each card below shows one RPG II specification type laid out across all 80 columns, styled as a punched card. Colored bands mark every field region; the label above each band names the field, and the numbers beneath give the exact column range. Tick marks at the top edge show every column (longer ticks at every 5th). A worked example line sits at the bottom of each card, aligned column-by-column so you can see how real source code maps onto the layout. The reference table under each card lists the valid entries for every field.

I-spec · Input Specifications

Maps record positions to named fields. Record-identification lines name the file; field-description lines declare each field’s position, type, and name.

Record-identification lines use columns 7–20 (filename + record ID). Field-description lines use columns 43–70. Both share column 6 (form type I).

O-spec · Output Specifications

Describes printed output declaratively. Record lines set line type and spacing; field lines name each output field or constant.

Record lines use columns 7–31 (filename, type, spacing, conditioning). Field lines use columns 32–70 (field name, edit code, end position, constant). Both share columns 23–31 (conditioning indicators).
