Chapter 17 — Output Formatting

↑ Table of contents

Summary

RPG II formats numeric output through two mechanisms on the O-spec: edit codes (a single-letter code in column 38 that selects a predefined format for commas, decimal points, signs, and zero-suppression) and edit words (a quoted template string in columns 45–70 that gives you full control over the output picture). Edit codes are quick — like choosing a format specifier. Edit words are expressive — like writing a custom format template. This chapter covers all edit codes (1–4, A–D, J–Q), edit word syntax (replaceable characters, zero-suppression stop, check protection, sign suffixes), and the zero-balance blank behavior that distinguishes adjacent code pairs.

Outline

17.1 The "you already know" analogy

  • Edit codes are like format specifiers — String.Format("{0:N2}", value) or Python's f"{value:,.2f}". One code selects a whole formatting pattern.
  • Edit words are like custom format templates — "$#,###.00" or "__.__". You draw the picture; the runtime fills in the digits.

The pairing rule: adjacent codes differ only in zero-balance behavior. Code 1 and 2 are identical except that 2 blanks out a zero value. Same for 3/4, A/B, C/D, J/K, L/M, N/O, P/Q.

17.3 Edit words (cols 45–70)

  • When column 38 is blank and columns 45–70 contain a quoted string, the string is an edit word — a template the runtime fills with the field's digits.
  • Edit word syntax:
    • Blank = replaceable position (filled by source digits, right-aligned, zero-suppressed).
    • Comma, period, slash, etc. = literal characters printed as-is.
    • First 0 = stops zero-suppression (digits before it always print, even zeros).
    • Asterisk * = check protection (replaces leading zeros with asterisks instead of blanks).
    • Trailing - or CR = sign indicator (printed only if the value is negative; blank otherwise).
    • & = forces a literal blank in the output.
  • Example: edit word ' , .' with value 1234 → " 1,234" (digits filled right, comma inserted, leading blanks suppressed).

17.4 When to use edit codes vs edit words

  • Edit codes for standard formatting (commas, decimals, optional sign). Quick — one letter does the job.
  • Edit words when you need custom literal text interspersed with digits — phone numbers, dates, currency symbols, or non-standard separators.
