Chapter 12 — Loops and Branching

↑ Table of contents

In this chapter

RPG II gives you three loop constructs and one branching mechanism. The loops are: DOWxx (do-while — test at the top, like a modern while), DOUxx (do-until — test at the bottom, like a modern do-while), and DO (a counted loop with start, limit, and increment — like a modern for). The branching mechanism is GOTO/TAG: an unconditional jump to a labeled position. All three loops close with END and reuse the same six comparison operators (GT/LT/EQ/NE/GE/LE) you met in Chapter 11. This chapter covers each construct, when the body runs (zero-or-more vs. one-or-more times), nesting, and the patterns for early exit.

12.1 The "you already know" analogy

You already know three kinds of loop from every language you have used: a pre-test loop (while), a post-test loop (do-while), and a counted loop (for). RPG II maps onto them directly:

  • DOWxx = while loop. The test comes first; the body may run zero times.
  • DOUxx = do-while loop. The body runs first; the test comes at the bottom; the body always runs at least once.
  • DO = for loop with a start value, a limit, and an increment.
  • GOTO/TAG = goto and label — yes, RPG II has them, and they are genuinely useful for early-exit patterns.

The one twist. In a modern language you write the loop condition inside parentheses: while (cnt < 5). In RPG II the comparison is embedded in the operation name itself — DOWLT means "do-while less-than." Factor 1 and Factor 2 are the two operands being compared, exactly as in IFxx.

12.2 DOWxx — do-while (test at top)

DOWxx tests the condition before each iteration. If the condition is false on the very first test, the body never runs at all — zero iterations. The body executes only while the condition stays true.

Factor 1 and Factor 2 are the two operands; the two-letter suffix picks the operator. DOWLT means "do while less-than," so CNT DOWLT 5 reads as "while CNT < 5, run the body." Here is a complete loop that sums the integers 1 through 5:

The loop repeats until CNT reaches 5, at which point 5 < 5 is false and the loop exits. SUM holds 1+2+3+4+5 = 15.

12.3 DOUxx — do-until (test at bottom)

DOUxx runs the body first, then tests at the bottom. The body always runs at least once. The loop exits when the condition becomes true — note this is the opposite sense from DOWxx, which continues while true. Think of it as "do the body until the condition is met."

DOUGE fills cols 28–32; 3 is the limit at col 33. The body runs, then END tests whether CNT >= 3. If not, it loops back; if so, it exits. Trace: CNT starts at 0, the body runs unconditionally (CNT=1, SUM=1), then tests 1 >= 3 — false, loop again. Second iteration: CNT=2, SUM=2, test false. Third: CNT=3, SUM=3, test true — exit. SUM = 3.

DOW vs. DOU — when the body runs. DOWxx tests first, so the body may run zero times (if the condition starts false). DOUxx runs the body first, so it always runs at least once. If you need "read a record, then check if it's the one I want," use DOUxx — you need at least one read before you can test.

12.4 DO — counted loop

DO is the counted loop — RPG II's for. Factor 1 is the starting value, Factor 2 is the limit, and the result field (cols 43–48) is the index variable. The loop sets the index to the start value, runs the body while the index is ≤ the limit, and at END adds the increment. The default start is 1 and the default increment is 1.

Here 1 is at col 18 (Factor 1 = start), DO at cols 28–29, 5 at col 33 (Factor 2 = limit), and IDX at col 43 (result = the index). The body runs for IDX = 1, 2, 3, 4, 5. SUM = 15.

12.5 GOTO and TAG

TAG marks a position in the code — it does nothing except serve as a jump target. The label name goes in Factor 1 (cols 18–27). GOTO jumps unconditionally to the matching TAG; the target label goes in Factor 2 (cols 33–42). Both forward and backward jumps work.

This is a hand-rolled loop. LOOP TAG plants the label. The body increments CNT and adds it to SUM. The COMP sets indicator 99 in the HI slot (cols 54–55) when CNT > 5. The GOTO is conditioned on N99 (cols 9–11: indicator 99 OFF), so it jumps back to LOOP only while CNT has not exceeded 5. Once CNT reaches 6, indicator 99 turns ON, the N99 condition fails, and execution falls through past the GOTO. SUM = 1+2+3+4+5 = 15.

This is familiar ground. TAG is a label; GOTO is goto. The common use is early exit from a loop body — jump to a TAG after the END to break out. RPG II has no BREAK or CONTINUE operation, so GOTO is how you emulate them.

12.6 Nesting and END

Loops and IFxx blocks can nest arbitrarily deep. Each level needs its own END, and the compiler matches each END to the innermost open block — just like closing braces in a modern language.

12.7 The xx comparison operators

DOWxx and DOUxx use the same six two-letter operators as IFxx. The table below shows what each means in both loop forms — note again the opposite exit sense:

12.8 Which loop to choose

  • DO when you know the count up front — iterating a fixed number of times, walking an array by index. Closest to a for loop.
  • DOWxx when the body might not need to run at all — "while there is more data, keep going." Closest to a while loop.
  • DOUxx when the body must run at least once before you can test — "process one item, then check whether to stop." Closest to a do-while loop.
  • GOTO/TAG for early exit from a loop (there is no BREAK), or for flow that does not fit the three structured forms.

Common pitfalls

  • No BREAK or CONTINUE. To exit a loop early, GOTO a TAG placed after the END. To skip the rest of an iteration, GOTO the END itself.
  • Five-character opcodes pack tightly. DOWLT, DOUGE, and DOWLE fill cols 28–32 completely, so Factor 2 begins immediately at col 33 with no gap — e.g. DOWLT5. This is correct, not a typo.
  • DOU always runs once. If the condition is already true at the start, DOWxx skips the body but DOUxx still runs it once. Choose accordingly.
  • GOTO cannot cross a subroutine boundary. A GOTO may only target a TAG in the same scope (main body or the same subroutine). See Chapter 13.
